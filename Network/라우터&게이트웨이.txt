### Routing Protocol 종류 

- 정적 라우팅 (Static Routing)
관리자가 네트워크에 대한 경로 정보를 직접 지정하여 라우팅.

(장점) 관리자에 의한 라우팅 정보만을 참조하여 라우터 자체 부담이 줄고, 동적 라우팅 보다 빠르며 안정적.
(단점) 네트워크 변화가 빈번하거나 등록할 네트워크 수가 많을 수록 설정을 변경하기 어려움

- 동적 라우팅 (Dynamic Routing)
대규모 네트워크에 사용하며 라우터 간의 변경된 네트워크에 대한 정보를 자동으로 교환하여 라우팅

(장점) Routing Table.을 자동으로 작성하여 관리자의 초기 설정만 필요.
(단점) Static Routing 보다 메모리를 많이 차지함

### Router 기능
OSI 7계층 중 3계층에 속하는 장비.

- 경로 설정 (path determination)
데이터 패킷이 목적지까지 갈 수 있는 길을 검사하고 어떤 경로로 전송하는 것이 가장 효율적일지 결정함

- 스위칭 (Switching)
경로가 결정되면 해당 경로로 데이터 패킷을 넘겨주는 것을 말함


* 게이트웨이
- 이중 프로토콜 및 네트워크 간에 통신을 가능하게 하며, 다른 네트워크로 들어가는 문 역할하는 네트워크 포인트
- 하나 이상의 프로토콜을 사용하는 측면에서 라우터와 다르다.

** Router : OSI 3계층 및 4계층 / 게이트웨이 : OSI 5계층 이상
** Router : 하드웨어 / 게이트웨이 : 소프트웨어

* 프로세스(process) : 프로그램이 메모리에 올라가서 실행 중인 것\
* 스레드(thread) : 프로세스의 실행 단위를 스레드
